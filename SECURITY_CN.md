# 安全策略

[English](./SECURITY.md)

BitFun 是一个运行在你本机的桌面级 Agent 运行时（Rust 内核 + Tauri 外壳），具备文件系统、终端、Git、MCP/LSP 以及远程控制等广泛能力。正因为权限范围较大，我们非常重视安全问题，并感谢社区帮助我们保护用户安全。

## 受支持的版本

BitFun 目前处于 `0.x` 活跃开发阶段，采用滚动发布。安全修复只会合入最新发布版本，旧版本不会单独打补丁。

| 版本 | 是否支持 |
| ---- | -------- |
| 最新发布版（`main`） | ✅ |
| 较旧的发布版 | ❌ |

在上报问题前，请先升级到最新[发布版](https://github.com/GCWing/BitFun/releases)，确认问题仍可复现。

## 如何上报漏洞

**请不要通过公开的 Issue、Discussion 或 Pull Request 来上报安全漏洞。** 在修复发布之前公开披露会让用户面临风险。

请通过 GitHub Security Advisories 私密上报：

➡️ **[上报漏洞](https://github.com/GCWing/BitFun/security/advisories/new)**

该入口会开启一个仅维护者可见的私密渠道。如果你无法使用 GitHub Security Advisories，可以新建一个极简的公开 Issue，仅写明「我想私下上报一个安全问题」——不要附带任何细节——维护者会与你建立私密沟通渠道。

为便于我们快速分诊，请尽量提供以下信息：

- 漏洞的清晰描述及其影响
- 受影响的组件（Rust 内核、桌面端/Tauri、Web UI、移动端配对、服务端/中继、CLI、安装器等）
- 复现步骤或概念验证（PoC）
- 受影响的版本、操作系统及相关配置
- 如有缓解措施或修复建议，也欢迎一并提供

## 披露流程

- 我们会争取在 **5 个工作日内**确认收到新的上报。
- 我们会与你一起确认问题、评估严重程度并确定修复时间线，并持续向你同步进展。
- 修复发布后，我们会发布安全公告，并在你愿意的情况下致谢上报者（若你希望匿名，我们将予以尊重）。
- 我们遵循协调披露原则：请在公开披露前给我们留出合理的修复窗口期。

## 适用范围

适用范围内：

- 本仓库中的 BitFun 运行时、官方 Agent、桌面端/CLI/服务端应用、Web UI，以及移动端配对/远程控制流程。

不在适用范围内：

- 第三方依赖中的问题（请向上游上报；如需 BitFun 侧更新请告知我们）。
- 需要事先攻陷机器、物理访问或已提权环境才能触发的漏洞。
- 运行自主 Agent 时由你明确授予的能力所固有的风险（例如某个你已授权的工具在其权限范围内执行操作）。

## 安全港（Safe Harbor）

对于善意发现并按照本策略上报漏洞，且在测试过程中避免侵犯隐私、破坏数据与中断服务的研究者，我们不会对其采取或支持任何法律行动。

感谢你帮助保障 BitFun 及其用户的安全。

## Taiji 模块安全

### 1. Taiji 模块安全边界

- **数据源（CTP / Replay / TTS）的安全假设**：
  - CTP 数据馈送假定来自已授权的经纪商连接。taiji 模块不验证 CTP 凭证，认证工作交给上游 CTP 适配器。
  - Replay 数据被视为可信的本地输入。来自不受信任来源的回放文件在导入前必须经过清洗。
  - TTS（时间刻度模拟）数据是合成数据，不依赖外部信任，但参数篡改可能扭曲回测结果。

- **策略引擎的沙箱边界**：
  - 用户自定义策略在进程内运行，不受沙箱隔离。策略代码拥有与 taiji 运行时相同的操作系统权限。执行第三方策略代码前务必审查。
  - 策略的网络访问默认不受限制。需要隔离的策略应在专用容器或虚拟机中执行。

- **实时数据的传输加密要求**：
  - 生产环境中，通过 TCP/UDP 传输的实时行情数据应使用加密通道（TLS）。明文 CTP 连接仅允许在隔离的测试网络中使用。

### 2. Taiji 依赖的安全策略

- **非 crates.io 依赖的审计流程**：
  - 任何来自 crates.io 之外的依赖（Git 仓库、本地路径、供应商 fork）在引入前必须经过人工审计。审计结果记录在依赖的提交历史或受跟踪的决策日志中。
  - 将非 crates.io 依赖锁定到具体的 commit hash，禁止使用浮动分支。

- **ffmpeg-sidecar 的安全注意事项**：
  - `ffmpeg-sidecar` 会在运行时下载预编译的 ffmpeg 二进制文件。首次用于生产前，请对比二进制文件的校验和与可信基线。
  - 如果运行时下载路径不适合你的环境，请在 taiji 发行版中捆绑一个已知安全的 ffmpeg 二进制文件。

### 3. 敏感信息处理

- **策略参数与 API 密钥**：
  - 策略参数、API 密钥和经纪商凭证始终优先使用环境变量。禁止在策略源文件、YAML 配置中硬编码密钥，也禁止将其提交到版本控制。
  - 本地开发使用 `.env` 文件（通过 `.gitignore` 排除在 Git 之外）。生产环境通过平台的密钥管理机制注入密钥。
  - 日志输出严禁包含完整的 API 密钥或凭证字符串。敏感字段在写入日志前必须脱敏。
